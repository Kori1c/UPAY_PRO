# UPAY PRO 全面排查问题清单 (Deep Audit)

在您的要求下，我暂时停止了修复动作，对项目的后端架构、数据库设计、并发处理以及前端工程化进行了“地毯式”的深度审查。

以下清单用于标记**仍需重点关注或需要重新确认的隐患**。其中一部分问题已经在当前分支中得到修复或部分缓解，因此这里更适合作为“持续复核清单”，而不是“全部仍然未修”的绝对结论。

## 🔴 严重架构与稳定性隐患 (Critical)

- [~] **1. 扫块程序存在 `log.Fatalf`，会引发进程暴毙**
  - **位置**: `BSC_USD/bsc_scanner.go` 等多个扫块模块的 `Start_scan` 函数。
  - **当前状态**: 当前主分支中可直接检索到的 `log.Fatalf` 基本已经移除或注释掉，扫块异常更多改为记录日志继续运行。
  - **剩余关注点**: 仍建议继续按链别逐个复查 scanner 分支实现，避免隐藏在低频路径中的直接退出逻辑。

- [~] **2. Redis 配置存在“冷启动陷阱” (设计缺陷)**
  - **位置**: `db/rdb/rdb.go` (L16-L62) 和 `web.go` 中的设置保存。
  - **当前状态**: 当前分支已经补上 `rdb.Reload()` 和 `mq.Reload()`，系统设置保存后会尝试重载 Redis 与任务队列，因此“必须重启才生效”的问题已被明显缓解。
  - **剩余关注点**: 仍需继续验证异常回滚、极端配置错误和长连接切换时的稳定性。

- [x] **3. `CreateTransaction` 的全局锁瓶颈已被移除**
  - **位置**: `web/function.go`
  - **当前状态**: 当前主下单路径已经不再依赖粗粒度的全局 `sync.Mutex`。现状改为：
    - 基于 `order_id` 的 Redis `SETNX` 并发锁，避免同一商户订单号被重复并发创建。
    - 基于 `token + actual_amount` 的 Redis 预占，避免同钱包同金额被重复分配。
  - **剩余关注点**: 这条高风险问题目前可视为已完成第一阶段修复，但仍建议后续补充更系统的并发压测，验证极端高并发下的吞吐和冲突回退体验。

- [x] **4. 异步回调缺乏重试与容错队列**
  - **位置**: `cron/cron.go` (L436-L437)
  - **当前状态**: 当前分支中的通知发送已走 `dispatchNotificationWithRetry`，带有多次延迟重试，不再是单次“发射后不管”。

## 🟠 业务逻辑与中间件缺陷 (High)

- [x] **5. JWT 鉴权中间件错误地使用 302 重定向**
  - **位置**: `web/function.go` (L101-L144) 的 `JWTAuthMiddleware`
  - **当前状态**: 当前分支里 `JWTAuthMiddleware` 已区分 `/admin/api/*` 与普通页面路由。API 未登录时返回 JSON `401`，普通页面仍跳转登录页。

- [x] **6. JWT 生成失败时触发 Panic**
  - **位置**: `web/function.go` (L63) 的 `GenerateToken`
  - **当前状态**: 当前分支中 `GenerateToken` 已改为返回 `(string, error)`，不再直接 `panic`。

- [x] **7. 订单回调失败次数更新目标已修正**
  - **位置**: `cron/cron.go`
  - **当前状态**: 当前分支里失败次数累加已经改为基于最新订单对象 `v1` 更新，不再使用旧副本 `v`。
  - **备注**: 这一项不再是当前阻塞问题，更值得继续关注的是回调重试节奏和外部通知通道的可观测性。

- [~] **9. TRON 扫链对外部限流仍需持续观察**
  - **位置**: `tron/TronGrid.go`、`tron/tron.go`、`cron/cron.go`
  - **当前状态**: 现已确认之前的 `429` 主要来自待支付 TRON 订单每 10 秒同时轮询 Tronscan 和 TronGrid。当前分支已经加入 TronGrid 限流冷却窗，收到 `429` 后会临时跳过后续请求，避免持续硬撞接口。
  - **剩余关注点**:
    - 这次属于“止血型”缓解，不是最终形态。
    - 后续仍建议结合订单量、API key 配额和轮询频率，继续评估是否要引入更细的链路分层、动态退避或优先级策略。

## 🟡 前端工程化问题 (Medium)
